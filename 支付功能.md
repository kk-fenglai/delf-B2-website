# 订阅支付功能实现总结

DELF B2 平台的订阅支付闭环：用户在 Pricing 页选择套餐 → 扫码支付 → 渠道回调入账 → 自动续费到期扣款。**v1 国内双通道**（微信支付 V3 + 支付宝），**支持签约代扣**，**超管后台手动退款**，价目在数据库可配置。

> **2026-04 立项背景**：之前的 Pricing 页是纯展示，前端按钮跳回自身；Stripe 只在 PRD 里写了没落地。国内用户占比高，优先做微信 + 支付宝，Stripe 留到 v2.0 出海时再加。

## 一、后端实现

### 1. 数据模型 `backend/prisma/schema.prisma`

| 模型 | 职责 |
|---|---|
| `Product` | 套餐（STANDARD / AI / AI_UNLIMITED），一对多 `Price` |
| `Price` | 价格档（`STANDARD_1M` / `STANDARD_12M` …），`months` + `amountCents` + `supportsAutoRenew` |
| `PaymentOrder`（扩展） | 订单主表；加 `priceId`、`contractId`、`refundedCents`；加 `@@unique([provider, providerOrderNo])` 防重放 |
| `Subscription`（已存在） | 开通历史，一次购买一行；`sourceOrderId` 关联订单 |
| `PayContract`（新） | 签约代扣合约；`externalContractId` + `nextChargeAt` + `failedCount`；3 次失败转 `SUSPENDED` |
| `RefundOrder`（新） | 退款单；`externalRefundNo` 唯一 |

迁移：`backend/prisma/migrations/<ts>_billing_v2/`（不改 `20260423154430_billing`，向前追加）。

### 2. 服务层

- **`services/billing.js`**：`applyPurchaseToUser({userId, plan, months, sourceOrderId})` 延长 `subscriptionEnd` + 写 `Subscription`；新增 `revokePurchase(orderId)` 处理全额退款回落
- **`services/payments/wechat.js`**：封装 wechatpay-axios-plugin，导出 `createNativeOrder` / `queryOrder` / `closeOrder` / `verifyNotify` / `decryptResource` / `refund` / `signContract` / `payByContract`
- **`services/payments/alipay.js`**：封装 alipay-sdk，导出同构的 `createPrecreate` / `tradeQuery` / `tradeClose` / `verifyNotifySign` / `refund` / `signAgreement` / `agreementPay`
- **`services/payments/reconcile.js`**：单进程 `setInterval(10min)` worker，三职责合一：
  - **关单**：`PENDING AND expiresAt < now` → 调渠道 close → 置 `CLOSED`
  - **补单**：`PENDING AND createdAt > now-24h` → 调渠道 query → 若渠道已 SUCCESS 本地 PENDING，走 notify 相同入账（网络丢包兜底）
  - **代扣**：`PayContract.status=ACTIVE AND nextChargeAt <= now` → 渠道代扣 → 成功入账 + 推下期；失败累计

### 3. 路由

| Method | Path | 说明 |
|---|---|---|
| `GET` | `/api/pay/products` | 公开，读 Product + Price 列表（前端价目） |
| `POST` | `/api/pay/wechat/native` | 下单 + 返回 `code_url` |
| `POST` | `/api/pay/wechat/sign` | 签约链接（`/v3/papay/contracts/pre-entrust-web`） |
| `POST` | `/api/pay/wechat/unsign` | 解约 |
| `POST` | `/api/pay/wechat/notify` | V3 回调：支付结果 / 签约 / 代扣，一个入口按 event_type 分发 |
| `POST` | `/api/pay/alipay/create` | precreate_qr 或 page_pay |
| `POST` | `/api/pay/alipay/sign` | cycle agreement `alipay.user.agreement.page.sign` |
| `POST` | `/api/pay/alipay/unsign` | `alipay.user.agreement.unsign` |
| `POST` | `/api/pay/alipay/notify` | RSA2 异步回调 |
| `GET` | `/api/pay/orders` | 用户自己订单列表（分页） |
| `GET` | `/api/pay/orders/:id` | 单订单状态（1.5s 前端轮询） |
| `GET` | `/api/pay/contracts` | 当前用户合约（用于管理自动续费 UI） |

管理员：

| Path | 说明 |
|---|---|
| `/api/admin/products` `/api/admin/prices` | 商品/价格 CRUD |
| `GET /api/admin/payment-orders` | 订单对账表（筛选 status / provider / 时间范围） |
| `POST /api/admin/payment-orders/:id/refund` | 手动退款（`X-Admin-Password` 二次确认） |
| `GET /api/admin/contracts` + 强制解约 | 合约运营面板 |

### 4. 幂等 / 安全机制

- **唯一键**：`PaymentOrder(provider, providerOrderNo)` 联合 unique；notify 重复到达直接被 DB 挡掉
- **行级事务**：入账用 `updateMany({ where: { id, status: 'PENDING' }, data: { status: 'PAID' } })`，受影响行数 = 0 即视为已处理，返回渠道 `success`
- **金额**：全部 `Int(cents)` 存储，服务端依 `priceId` 重算，不信任前端入参
- **签名校验**：
  - 微信：`Wechatpay-Signature` + `Wechatpay-Timestamp` + `Wechatpay-Nonce` + 平台证书验签；body 过 AES-GCM 解密
  - 支付宝：按字段 ASCII 排序 + RSA2 签名 + 支付宝公钥
- **IP allowlist**：Nginx 层可选限制 notify 来源
- **rate limit**：`/create|native|sign` 10/min/用户；notify 不限速；admin refund 5/min
- **Worker 并发安全**：模仿 `essayQueue.js`，代扣 / 关单前先 `UPDATE ... WHERE status='ACTIVE' AND nextChargeAt <= now` 抢行，抢到才处理

### 5. 启停集成 `backend/src/index.js`

```js
const { startReconcileWorker } = require('./services/payments/reconcile');
// ...
app.use('/api/pay/wechat', payLimiter, wechatPayRoutes);
app.use('/api/pay/alipay', payLimiter, alipayRoutes);
app.use('/api/pay/orders', payOrderRoutes);
app.use('/api/pay/products', productRoutes);
// ...
startReconcileWorker();
// SIGTERM：reconcile.drain() + prisma.$disconnect()
```

`env.js` 生产启动校验：
- `IS_PROD && !WECHAT 配齐 && !ALIPAY 配齐` → `exit(1)`
- `PAY_PUBLIC_BASE_URL` 必须 `https://`
- `DEV + PAY_MOCK_ENABLED=true` 才挂 `/orders/:id/mock-pay`

### 6. 审计日志

所有支付写入点都走 `middleware/admin.js:writeAdminLog`：

| action | 触发 |
|---|---|
| `PAYMENT_COMPLETED` | notify 入账成功 |
| `PAYMENT_FAILED` | 验签失败 / 金额不符 / DB 写入异常 |
| `CONTRACT_SIGNED` | 签约回调成功 |
| `CONTRACT_TERMINATED` | 用户主动 / 渠道方解约 / 3 次失败 |
| `PAYMENT_REFUNDED` | 退款回调成功 |
| `PAYMENT_RECONCILE_FIXUP` | worker 补单命中 |

## 四、错误分类

| `code` | 场景 | 前端表现 |
|---|---|---|
| `PAY_NOT_CONFIGURED` | 生产环境未配齐渠道凭证 | 500 拦截 |
| `INVALID_PRICE` | `priceId` 不存在 / 已停用 | 400 |
| `AMOUNT_MISMATCH` | 回调金额 ≠ 订单金额 | 写 AdminLog + 拒绝入账 |
| `NOTIFY_SIGNATURE_INVALID` | 验签失败 | 返回渠道 FAIL + 写审计 |
| `DUPLICATE_NOTIFY` | 幂等命中 | 返回 SUCCESS 正常忽略 |
| `CONTRACT_SUSPENDED` | 代扣连续 3 次失败 | 邮件通知 + UI 提示 |
| `REFUND_AMOUNT_EXCEEDS` | 退款金额 > 剩余可退 | 400 |
| `REFUND_CHANNEL_FAILED` | 渠道 refund 调用失败 | 保留 PENDING 等人工 |

## 五、部署前必做

1. **安装依赖**
   ```bash
   cd backend && npm install wechatpay-axios-plugin alipay-sdk
   ```
2. **执行迁移**
   ```bash
   unset DATABASE_URL && cd backend && npx prisma migrate deploy
   ```
3. **seed 价目**
   ```bash
   cd backend && npm run seed:billing   # 新增脚本
   ```
4. **配齐 `.env`**（至少一套渠道 + 公共）：
   ```
   WECHAT_APP_ID / WECHAT_MCHID / WECHAT_SERIAL_NO
   WECHAT_APIV3_KEY / WECHAT_PRIVATE_KEY_PEM / WECHAT_PLATFORM_CERT_PEM
   ALIPAY_APP_ID / ALIPAY_PRIVATE_KEY_PEM / ALIPAY_PUBLIC_KEY_PEM
   PAY_PUBLIC_BASE_URL=https://api.your-domain.com
   ```
   自动续费（签约 + 代扣）额外需要：
   ```
   WECHAT_PAPAY_PLAN_ID=xxxx            # 微信商户平台配置的周期扣款模板 plan_id
   ALIPAY_PERIOD_RULE_JSON={...}        # 支付宝 period_rule_params 的 JSON 字符串
   ```
5. **渠道商户后台**：把 notify URL 配成 `https://api.your-domain.com/api/pay/{wechat,alipay}/notify`
6. **Nginx**：放行 `/api/pay/*/notify` 不走 auth、不限速、但走 WAF；限制 body ≤ 1 MB
7. **CSP**：`backend/src/index.js` 的 `connectSrc` / `frameSrc` 视需要加支付宝收银台域名（若用 page_pay）

## 六、验证清单

### 本地（dev，mock 模式）

- 登录 → `/pricing` → 买 STANDARD → 「模拟支付成功」→ `/me` 看 `plan=STANDARD`
- `/orders` 列一行 PAID
- admin → 改 AI 价 → 前端刷新价目可见
- admin 退款 → `/me` 回落 FREE + `refundedCents` 正确

### 沙箱（微信 + Alipay）

- 真实下单 → 手机扫码 → 回调到账 → 订单 PAID
- 故意等 15 min 不付 → worker 置 CLOSED
- 断后端接收 notify，10 min 后 worker query 应补单
- sign → 沙箱签约成功 → 合约 ACTIVE → 手动把 `nextChargeAt` 提前 → worker 扣款成功 → `subscriptionEnd` 延长
- 无余额测试号 → 连扣 3 次失败 → 合约 SUSPENDED + 邮件

### 生产灰度

1. Pricing 页 A/B：5% 流量开启真实支付
2. 7 天监控 `AdminLog` 里：`PAYMENT_FAILED` / `CONTRACT_SUSPENDED` / `PAYMENT_RECONCILE_FIXUP` 数量
3. `PENDING > 30 min` 订单数应接近 0
4. 无异常后 50% → 100%

## 十、接入代办（你如何引入微信/支付宝）

> 目的：把“你拿到的商户资料/证书/密钥”落到 `.env`，并在微信/支付宝后台把回调 URL 配对，最后按验证清单跑通。

### A. 微信支付 V3（扫码 + 周期扣款）

1. **准备商户资料**
   - 商户号 `WECHAT_MCHID`
   - 商户 API 证书序列号 `WECHAT_SERIAL_NO`
   - 商户私钥（PEM，多行）`WECHAT_PRIVATE_KEY_PEM`
   - APIv3 密钥（32 bytes）`WECHAT_APIV3_KEY`
   - 微信平台证书（PEM，多行）`WECHAT_PLATFORM_CERT_PEM`
2. **配置回调**
   - 在微信商户平台配置支付回调：`/api/pay/wechat/notify`
3. **启用周期扣款（自动续费）**
   - 商户平台申请开通周期扣款能力（Papay）
   - 配置周期扣款模板并拿到 `plan_id` → 写入 `WECHAT_PAPAY_PLAN_ID`
4. **把配置写入 `.env`**
   - 同时确保 `PAY_PUBLIC_BASE_URL` 是公网可访问的 `https://...`

### B. 支付宝（扫码 + 代扣协议）

1. **准备商户资料**
   - 应用 APPID：`ALIPAY_APP_ID`
   - 应用私钥（PEM，多行）：`ALIPAY_PRIVATE_KEY_PEM`
   - 支付宝公钥（PEM，多行）：`ALIPAY_PUBLIC_KEY_PEM`
2. **配置回调**
   - 在支付宝开放平台配置异步通知：`/api/pay/alipay/notify`
3. **代扣协议（自动续费）**
   - 确认你账号已具备 `GENERAL_WITHHOLDING` / 周期扣款能力
   - 按业务周期产出 `period_rule_params`（JSON）→ 写入 `ALIPAY_PERIOD_RULE_JSON`
4. **把配置写入 `.env`**
   - 同样需要 `PAY_PUBLIC_BASE_URL=https://...`

### C. 上线前验证（建议顺序）

- **先扫码**：微信/支付宝各下 1 单，确认 notify 可达、订单入账、`/orders` 可见
- **再签约**：走 `/api/pay/{wechat,alipay}/sign`，签约成功后 `PayContract` 变 `ACTIVE`
- **再代扣**：把合约 `nextChargeAt` 手动改到当前时间前，等待 `reconcile` 扣款；成功后 `subscriptionEnd` 延长，合约 `nextChargeAt` 推到下期
